UI System: Controls & Architecture

Session 07 — The two-layer UI system: legacy UO controls + ImGui overlay

Overview

TazUO has two coexisting UI systems:

  1. Legacy UO Control System — Every visible window ("gump") is a Gump subclass composed of Control leaf nodes. This is the system the Ultima Online server communicates with.
  2. ImGui Overlay — Modern developer-tools-style windows using ImGuiNET. Used for the script manager, performance tools, and developer debugging.

Both systems run inside GameScene and render during the UI draw pass (after all game world rendering). UIManager (in Game/Managers/) is the static hub that manages the legacy system.

GameScene.Draw() ├── DrawWorld (4 render passes — see doc 04) └── DrawUI ├── UIManager.PreDraw() — layout recalculation ├── UIManager.Draw() — draw all Gumps (front to back) └── ImGui.Render() — draw ImGui overlay on top

UIManager

UIManager is a static class that owns the linked list of all active Gump instances. It handles hit-testing, drag & drop, keyboard focus routing, modal state, and the type-indexed gump cache.

Z-Ordering and Sort

Gumps are stored in a LinkedList. "In front" = at the head (index 0 = first). When clicked, a gump is promoted to the front via MakeTopMostGump(). The optional sort step (SortControlsByInfo()) runs when _needSort is set, ordering gumps by their UILayer enum value:

UILayerValueExamples
Default0All normal gumps
Over1Tooltips, popups
GameCursor2Item-drag cursor overlay

Modal Gumps

Any gump with IsModal = true prevents mouse clicks from reaching the world. If ModalClickOutsideAreaClosesThisControl = true, clicking outside the modal gump's bounds disposes it. Use sparingly — the player cannot interact with anything else while a modal is open.

Drag Behavior

Ctrl-held drag activates precision mode: movement is halved (CTRL_DRAG_SPEED_MULTIPLIER = 0.5f) and after 10 pixels of movement, the axis is locked to horizontal or vertical. This lets players precisely position gumps without accidentally moving in two dimensions.

Control — The Base Class

Control is the abstract base for every UI element. It owns position, size, children, input flags, and the draw lifecycle. All gumps, buttons, labels, and scroll areas are Controls.

Coordinate System

All positions are relative to the parent:

ScreenCoordinateX = ParentX + X    // absolute screen X
ScreenCoordinateY = ParentY + Y    // absolute screen Y
ParentX = Parent.X + Parent.ParentX  // recursive sum up to root

Key Properties

The Draw Lifecycle

UIManager.Update() └── foreach Gump → gump.Update() └── foreach child → child.Update() UIManager.PreDraw() └── foreach Gump → gump.PreDraw() // layout recalculation (ScrollArea uses this) UIManager.Draw(batcher) └── foreach Gump (Last → First) └── gump.Draw(batcher, x, y) └── foreach child (0 → n) └── child.Draw(batcher, x + child.X, y + child.Y)

Virtual Methods to Override

MethodWhen called
Draw(batcher, x, y)Every frame when visible. Base implementation draws all children. Override to add custom rendering before/after calling base.
Update()Every frame. Base calls children. Use for animation, state polling.
PreDraw()After Update, before Draw. Use for layout calculations that affect draw (ScrollArea recalculates scrollbar max here).
Contains(x, y)Hit-test: is screen point inside this control? Base checks bounds rectangle.
Dispose()Cleanup. Sets IsDisposed = true, disposes children.
AlphaChanged(old, new)Called when Alpha property changes. Override to update cached hue vectors.

Gump — The Window Base Class

Gump : Control is the base for every top-level UI window. It adds server/local serial association, save/restore functionality, lock behavior, and alpha-scroll support.

Serial Scheme

SerialPurpose
LocalSerialClient-assigned ID. For client-created gumps, often 0 or a made-up constant. For item-bound gumps, the item serial.
ServerSerialThe server's ID for this gump dialog. Packet-created gumps set this from the server's gump serial.
IsFromServerFlag indicating the gump was created by a server packet (generic gump). Server gumps can be closed by server command.

GumpType and Persistence

Each concrete gump subclass declares its GumpType enum value. When the game session is saved (logout), UIManager iterates all gumps that have CanBeSaved = true and calls Save(XmlTextWriter):

public virtual void Save(XmlTextWriter writer)
{
    writer.WriteAttributeString("type", ((int)GumpType).ToString());
    writer.WriteAttributeString("x", X.ToString());
    writer.WriteAttributeString("y", Y.ToString());
    writer.WriteAttributeString("serial", LocalSerial.ToString());
    writer.WriteAttributeString("isLocked", isLocked.ToString());
    writer.WriteAttributeString("alphaOffset", AlphaOffset.ToString());
}

On next login, Restore(XmlElement) is called with the saved XML to reapply position, lock state, and alpha. Gumps that should not be saved (transient dialogs, tooltips) override ShouldBeSaved => false.

Lock / Unlock

Ctrl+Alt+Click (or left trigger on controller) on any gump toggles its lock state. When locked: CanMove = false, CanCloseWithRightClick = false. The gump is visually immovable and protected from accidental closure.

Alpha Scrolling

Alt+ScrollWheel on any gump adjusts Alpha for the gump and all children by ±0.02, clamped to [0.1, 1.0]. This lets players make windows transparent. The AlphaOffset field stores the cumulative offset so it can be restored across sessions.

InvalidateContents Pattern

Heavy gumps (status bars, durability panels) use a deferred-rebuild pattern:

// Signal "I need to rebuild my content on the next Update()"
gump.RequestUpdateContents();   // sets InvalidateContents = true

// Gump.Update() calls this when InvalidateContents is true
protected virtual void UpdateContents() { }   // override to rebuild children

Button

Button renders three UO gump art images: _normal (default), _pressed (mouse down), _over (mouse hover). Size is inferred from the normal image at construction time.

new Button(
    buttonID:  0,       // ID sent to server (ButtonAction.Activate) or page switch number
    normal:    0x0481,  // gumpart ID for default state
    pressed:   0x0482,  // gumpart ID when held down
    over:      0x0483,  // gumpart ID when hovered (0 = same as normal)
    caption:   "OK",   // optional text overlay
    normalHue: 0x7FFF  // caption hue (ushort.MaxValue = no caption)
)
{
    ButtonAction = ButtonAction.Activate, // Activate=fire click; SwitchPage=change gump page
    X = 100, Y = 50
}
PropertyMeaning
ButtonIDPassed to server on click (GenericGump response) or used as page number
ButtonActionActivate = fire click event; SwitchPage = change parent's ActivePage
FontHue / HueHoverCaption color in normal / hover states
GumpPicBase.Contains() — pixel-perfect hit testing Hit testing for gump-art images checks GumpartLoader.PixelCheck() against the actual pixel data, not just the bounding rectangle. Transparent pixels in the art do not register clicks. This is how the irregular-shaped UO art buttons work correctly.

Label

Label wraps RenderedText, which pre-renders text into a texture on creation. Size is automatically set to the rendered text dimensions.

new Label(
    text:      "Hello World",
    isunicode: true,   // true = Unicode font; false = legacy ASCII font
    hue:       0x0481,
    maxwidth:  200,    // 0 = unlimited; wraps at maxwidth pixels
    font:      0xFF,   // 0xFF = use default font
    style:     FontStyle.BlackBorder,
    align:     TEXT_ALIGN_TYPE.TS_LEFT,
    ishtml:    false
)

Setting the Text property recreates the texture. Changing Hue also recreates via CreateTexture(). This is O(n) in text length so avoid updating labels every frame — only update when the value changes.

The second Label constructor (taking parts, lines) is used by the server gump packet parser to create labels from server-sent data.

Text Input Controls

TazUO provides two text input systems:

StbTextBox uses the StbTextEditSharp library for text editing (cursor, selection, undo). The TextBox.cs wrapper adds scrolling for multiline content.

ScrollArea

ScrollArea is a clipped viewport with a scrollbar. Content is added as children; the scroll bar moves the content offset. The maximum scroll value is recalculated in PreDraw() from the total height of all children.

var area = new ScrollArea(x: 10, y: 10, w: 200, h: 400,
    normalScrollbar: true  // true=classic UO scroll bar; false=scroll flag icon
) { Parent = myGump };

new Label("Item 1", ...) { Parent = area };
new Label("Item 2", ...) { Parent = area };
PropertyMeaning
ScrollValueCurrent scroll offset in pixels
ScrollbarBehaviourShowWhenDataExceedFromView (auto-hide) or ShowAlways
ScissorRectangleCustom clipping rect; defaults to the control bounds

ModernScrollArea (TazUO addition) uses a flat ModernScrollBar instead of the classic UO scroll graphics and provides smoother scrolling behavior.

GumpPic

GumpPic extends GumpPicBase to render a single gump art image. Setting Graphic automatically looks up the image from GumpartLoader and sizes the control to match.

new GumpPic(x: 0, y: 0, graphic: 0x082C, hue: 0) { Parent = myGump };
// Width/Height auto-set from gump art dimensions

Key variant — EmbeddedGumpPic: accepts a raw Texture2D instead of a gump art ID; used for custom textures loaded from embedded resources.

Pixel-perfect hit testing: GumpPicBase.Contains() delegates to GumpartLoader.PixelCheck(). Transparent pixels do not register mouse-over/click events, which is essential for the non-rectangular UO art shapes.

AlphaBlendControl

AlphaBlendControl is a simple semi-transparent solid-color rectangle. It is the standard background for TazUO modern gumps:

new AlphaBlendControl(alpha: 0.7f)
{
    X = 0, Y = 0,
    Width = 300, Height = 200,
    Hue = 0,
    BaseColor = Color.Black,  // background color before alpha blend
    Parent = myGump
};

The hue vector is cached in hueVector and rebuilt whenever Hue or Alpha changes via AlphaChanged(). This avoids recomputing on every draw call.

ImGui Overlay System ImGui

TazUO includes an ImGui layer for developer and power-user windows. ImGui renders entirely separately from the legacy control system — it draws after the game world and legacy UI, appearing on top of everything.

ImGuiWindow Base Class

public abstract class ImGuiWindow : IDisposable
{
    public abstract void DrawContent();  // override to build the window

    public void Draw()   // called every frame by the ImGui render loop
    {
        if (!_isVisible || !_isOpen) return;
        if (ImGui.Begin(Title, ref _isOpen, _windowFlags))
            DrawContent();   // immediate-mode: build all widgets here
        ImGui.End();
        if (rightclickClose) Dispose();
    }
}

ImGUIComponents — Shared Widgets

ImGUIComponents.cs provides reusable ImGui widget wrappers used across multiple windows: colored buttons, tooltips, checkbox groups, input fields with validation, etc.

Coexistence with Legacy UI

Two separate renders ImGui and the legacy control system are entirely independent. ImGui windows do not participate in the legacy hit-test system, cannot be anchored to other gumps, and do not use UIManager.Add(). They live in a separate list managed by GameScene and are drawn after UIManager.Draw().

Event System

Controls receive events through overridable protected methods. UIManager routes SDL events from the mouse/keyboard statics through to controls:

OnMouseUp(x, y, button)Mouse button released
OnMouseClick(x, y, button)Complete press+release without dragging
OnMouseDoubleClick(x, y, button)Double-click
OnMouseWheel(delta)Scroll wheel

OnMouseExit()Mouse left control bounds

Wiring Event Handlers Externally

Controls expose MouseClick, MouseDoubleClick, etc. as public Action delegates (not C# events). These can be set directly:

var btn = new NiceButton(0, 0, 120, 30, ButtonAction.Activate, "Click Me");
btn.MouseUp += (sender, e) =>
{
    if (e.Button == MouseButtonType.Left)
        GameActions.Print("Clicked!");
};

Pattern: Adding a New Feature Gump

Here is the minimal pattern for a new client-created gump window:

public class MyFeatureGump : Gump
{
    public MyFeatureGump(World world) : base(world, localSerial: 0, serverSerial: 0)
    {
        // CanMove lets the player drag the window
        CanMove = true;
        CanCloseWithRightClick = true;

        // Build the layout
        Width = 300; Height = 200;

        // Background
        new AlphaBlendControl(0.7f) { X=0, Y=0, Width=Width, Height=Height, Parent=this };

        // Content
        new Label("My Feature", true, 0x07FF) { X=10, Y=10, Parent=this };

        // Close button
        var close = new NiceButton(10, 170, 80, 20, ButtonAction.Activate, "Close")
            { Parent = this };
        close.MouseUp += (_, _) => Dispose();
    }

    // Register a GumpType so the position is saved across sessions
    public override GumpType GumpType => GumpType.MyFeature; // add to enum
}

Showing the Gump

// From anywhere that has a World reference:
UIManager.GetGump<MyFeatureGump>()?.Dispose(); // close any existing instance
UIManager.Add(new MyFeatureGump(world));

Checklist for New Gumps

  1. Add the class to Game/UI/Gumps/
  2. Add its GumpType value to the GumpType enum (in GumpType.cs)
  3. Register it in the gump restore switch in GameScene (the block that maps enum → new Gump() on login)
  4. Override Save() and Restore() if the gump has state beyond position and alpha
  5. Set ShouldBeSaved = false if the gump is transient and should not persist across sessions
