TazUO Developer Documentation

Comprehensive technical reference for the TazUO game client — a feature-rich fork of ClassicUO, the open-source Ultima Online classic client written in C# / .NET 10.

Project Overview

TazUO is a fork of ClassicUO that adds quality-of-life features, a Python scripting engine, grid-based inventory management, modern UI overlays via ImGui, and deep customisation options requested by the community. The codebase is approximately 86,000 lines of C# across 579 source files, organised into five projects within a single Visual Studio solution.

TazUO vs ClassicUO
TazUO selectively merges upstream ClassicUO patches while maintaining its own feature set. If you see code that looks unfamiliar from ClassicUO, check the TazUO-specific managers under Game/Managers/ and the LegionScripting/ directory first.

Key Architectural Patterns

PatternWhere usedWhy it matters
Singleton via static class Client.Game, World, ProfileManager All systems share one instance; thread-safety is the caller's responsibility
Serial-keyed dictionaries World.Mobiles, World.Items Every in-game entity has a unique 32-bit serial; use World.Get(serial)
TNext linked list per tile Map/Chunk, render sorting Spatial lookup without allocations; CRITICAL — draw order must follow list order
Time.Ticks comparisons Every system that throttles updates uint millisecond counter; compare with > not ==
Profile JSON (snake_case) Configuration/Profile.cs Each property needs a [JsonSerializable] context or it won't round-trip
Snapshot before iteration AutoLootManager, HealthLinesManager Call .ToList() or World.GetCorpseSnapshot() before foreach loops
IsDestroyed guard Rendering, update loops Always check obj.IsDestroyed before accessing any GameObject field
Important: JSON serialisation
TazUO uses System.Text.Json with source-generated contexts. Every new type that needs serialisation must be added to the appropriate JsonSerializerContext subclass. Adding [JsonSerializable(typeof(MyType))] to the context is mandatory — the AOT compiler will not generate reflection-based fallbacks.
