ClassicAssist — Getting Started

TazUO bundles ClassicAssist alongside RazorEnhanced. This guide explains how to switch to it, what it offers, and how to get automation scripts running quickly.

What is ClassicAssist?

ClassicAssist is an open-source UO assistant by Reetus that plugs directly into the ClassicUO plugin API. It provides a full macro/scripting environment, item organiser, dress/undress agent, auto-follow, re-stock, and many other quality-of-life agents — all configured through its own dockable UI window inside the game.

Bundled, not default
ClassicAssist.dll is included in Data/Plugins/ but is not enabled by default. settings.json keeps "plugins": ["RazorEnhanced.dll"] so existing users are unaffected. You opt in via the in-game Plugins page.

Enabling ClassicAssist

  1. Open Options
    Press Alt+O (default keybind) or use the in-game menu to open the Options window.
  2. Navigate to Plugins
    Click the Plugins tab. You will see a checkbox row for every DLL found in Data/Plugins/.
  3. Check ClassicAssist, uncheck RazorEnhanced (optional)
    Tick ClassicAssist.dll. If you do not need Razor Enhanced, uncheck RazorEnhanced.dll. Both can run at the same time — see Running both plugins.
  4. Restart TazUO
    Plugin load/unload takes effect on the next game launch. Close and reopen TazUO.
  5. Open ClassicAssist
    After login the ClassicAssist window appears automatically. If it does not, look for its button in the bottom toolbar or type .classicassist in chat.
Persisted across sessions
Your plugin checkbox choices are saved back to settings.json under the "plugins" array. You only need to do this once per installation.

RazorEnhanced vs ClassicAssist

Both assistants implement the same ClassicUO plugin interface and are otherwise independent. Choose the one whose scripting model fits your workflow.

RazorEnhanced

  • Python 3 macros (IronPython)
  • Large existing macro library online
  • Agent-based UI (Dress, Organiser, Scavenger)
  • Familiar to long-time UO players
  • Default enabled in TazUO distributions

ClassicAssist

  • C# / Python macro engine
  • Modern WPF-style dockable UI
  • Built-in item property filter editor
  • Richer agent config (Dress, Restock, Follow, Vendor)
  • Active development & Discord community

First Steps in ClassicAssist

Macros tab

Click Macros in the ClassicAssist window. Press + to create a new macro. The built-in editor accepts Python 3. A minimal healing macro: 

# Heal self when HP drops below 60 %
if Hits() < MaxHits() * 0.6:
    UseSkill('Healing')
    WaitForTarget(3000)
    Target('self')

Agents tab

Pre-built agents handle repetitive tasks without scripting. Key agents and how to configure them:

AgentWhat it doesHow to configure
Dress Equips a saved outfit (armour, weapons, jewels) Drag items from your paperdoll onto the Dress agent slots → Save
Organiser Moves items between containers by graphic/hue/name Add item rules → set source/destination serial → Run
Restock Refills consumables from a bank/chest to your backpack Add items with desired quantity → target source container
Auto-Follow Automatically follows a named mobile Type target player name → Enable
Vendor Buy Automatically purchases items from vendor shops Add item rules → open a vendor → agent buys matching items

Profiles

ClassicAssist saves all its settings (macros, agents, hotkeys) into a profile JSON stored at:

%AppData%\ClassicAssist\Profiles\<profile-name>.json

You can keep separate profiles per shard or character and switch between them from the ClassicAssist Settings → Profiles menu.

Running Both Plugins Simultaneously

TazUO loads every DLL listed in settings.json → plugins in order. Both RE and CA can be active at once — they operate independently inside the shared ClassicUO plugin host.

Avoid double-automation
Running both plugins is fine, but activating the same agent type in both (e.g., two Dress agents or two auto-loot loops) will cause race conditions. Enable each agent in only one plugin at a time.

To load both, your settings.json plugins array should read:

"plugins": [
  "RazorEnhanced.dll",
  "ClassicAssist.dll"
]

This is set automatically when you check both boxes in the Plugins page and restart.

Developer / Build Setup

The packaging script (package.ps1 / package.bat) downloads the latest ClassicAssist release from GitHub automatically. For local development builds:

  1. Download ClassicAssist.dll from the latest ClassicAssist release.
  2. Place it at external/classicassist/ClassicAssist.dll in the repo root.
  3. Build normally — the CopyExternalDeps_build MSBuild target copies it to $(TargetDir)/Data/Plugins/ on every build.
Conditional copy
The .csproj ItemGroup for DataFiles_classicassist has a Condition="Exists(...)" guard. If the DLL is absent the build succeeds without it — nothing breaks in CI or on a fresh checkout.
PathWhen used
external/classicassist/ClassicAssist.dll Dev builds — dropped manually; copied by MSBuild
Data/Plugins/ClassicAssist.dll (in dist ZIP) End-user distributions built by package.ps1

Troubleshooting

SymptomLikely causeFix
ClassicAssist window never appears after login DLL not checked in Plugins page, or not in Data/Plugins/ Verify Data/Plugins/ClassicAssist.dll exists; check the Plugins page and restart
Game crashes on startup after enabling CA ClassicAssist DLL is built for a different .NET target than TazUO Ensure you downloaded the release build, not a debug/dev artifact; re-run package.ps1
package.ps1 prints "No ClassicAssist release ZIP found" GitHub API rate limit or release asset naming changed Re-run after a few minutes; if persistent, check the asset name filter in package.ps1 (*ClassicAssist*.zip)
Macro runs but has no effect Missing Pause() / Wait() between actions Add Pause(500) after actions that send packets; the server throttles rapid input
Both RE and CA loaded but RE macros stopped working Conflicting hotkey or agent overlap Check for duplicate hotkeys; disable any agent in CA that is also active in RE
Getting more help
The ClassicAssist Discord and GitHub Wiki are the best sources for macro API reference, agent how-tos, and community scripts. The TazUO Discord can help with plugin loading and packaging questions specific to this client.
